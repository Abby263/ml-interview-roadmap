## How to use this roadmap

This is the highest-coverage track in the platform. It is designed for candidates who need enough time to re-sequence fundamentals, deepen weak areas, and still finish with repeated mock practice.

The order matters:

- Days 1-12: statistics, probability, inference, experimentation, linear algebra, optimization, and evaluation math
- Days 13-28: traditional ML, feature engineering, leakage prevention, model selection, and ML coding
- Days 29-38: deep learning, CNNs, sequence models, attention, and transformers
- Days 39-50: MLOps, data validation, training pipelines, registry, CI/CD, serving, monitoring, and governance
- Days 51-62: GenAI, LLM basics, prompting, embeddings, vector search, RAG, fine-tuning, agents, and guardrails
- Days 63-72: LLMOps, prompt/model versioning, regression evals, tracing, routing, cost controls, privacy, safety, and red teaming
- Days 73-86: ML system design, feature stores, serving trade-offs, recommendations, search, ads, fraud, and RAG system cases
- Days 87-90: mocks, weak-area repair, project storytelling, and behavioral prep

Use the roadmap as a working schedule:

- complete the phase goal before moving on
- keep one implementation block and one speaking-practice block every week
- bookmark the topics that still feel slow or fuzzy after first review

## Exit criteria

By the final phase, you should be able to answer common ML theory prompts without notes, explain RAG and LLMOps trade-offs clearly, and design one ML system and one GenAI system end to end.
