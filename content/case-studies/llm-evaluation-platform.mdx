## Core objective

This system exists to stop silent regressions. That means it must track prompts, models, retrieval settings, and rubric outputs together, not as isolated artifacts.

## Main workflow

1. define evaluation suites by product or task
2. store prompt and model versions
3. run batch evaluations on a scheduled or release-triggered basis
4. score results with deterministic checks and rubric-based judges
5. surface regressions and block releases when thresholds fail

## Human review

Automated scoring helps scale. Human review is still important for safety-sensitive or subjective tasks. Mention sampling strategies instead of saying every run gets fully reviewed.
